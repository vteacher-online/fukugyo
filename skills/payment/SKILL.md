# paymentスキル

請求書の入金状況を管理し、未払いがあれば督促する。

## コマンド

```
/payment check        入金状況の一覧表示・期日超過アラート
/payment paid [ID]    入金済みとしてマーク
/payment remind [ID]  督促メールの文面を生成
```

---

## `/payment check` の表示

```
💰 入金状況 (2026-04-05 現在)

✅ 入金済み
  INV-2026-02-001  株式会社B  216,000円  2026-03-28入金

⏳ 入金待ち（期日前）
  INV-2026-03-002  株式会社B  216,000円  支払期限: 2026-04-30

🔴 期日超過
  INV-2026-03-001  株式会社A  432,000円  支払期限: 2026-04-30 → 5日超過
  → 督促メール（1回目）を生成しますか？ [y/n]
```

---

## 督促メールの3段階

### 1回目（期日超過直後〜7日）: やんわり確認

```
件名: 【ご確認】請求書のお支払いについて（INV-2026-03-001）

株式会社A
ご担当者様

いつもお世話になっております。山田太郎です。

2026年3月31日付でお送りした請求書（INV-2026-03-001、432,000円）について、
本日時点でご入金の確認ができておりません。

お手続き中でしたら恐れ入ります。
ご確認いただけますと幸いです。

ご不明な点がございましたらお気軽にご連絡ください。

山田太郎
```

### 2回目（1回目から7日後）: 明確に催促

件名に「【再送】」を付け、1回目を送った日付を明記する。
支払期限を改めて提示し、対応を明確に求める文面にする。

### 3回目（2回目から7日後）: 強い催促

入金がない場合の対応（法的措置の検討）に言及する文面にする。
`/escalate` スキルへの移行を提案する。

---

## 督促状況の記録

`invoices.json` に追記する：

```json
{
  "id": "INV-2026-03-001",
  "status": "督促中",
  "reminders": [
    { "sent_at": "2026-04-05", "level": 1 },
    { "sent_at": "2026-04-12", "level": 2 }
  ]
}
```

3回目の督促後も入金がない場合、`/escalate` の実行を強く促す。
